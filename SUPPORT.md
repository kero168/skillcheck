# Support

## Where to ask

- **Bug or unexpected behavior** — open a
  [bug report](https://github.com/kero168/skillcheck/issues/new?template=bug_report.yml)
  with the SKILL.md that reproduces it (or a minimized version).
- **Feature idea** — open a
  [feature request](https://github.com/kero168/skillcheck/issues/new?template=feature_request.yml).
- **Question / usage help** — use
  [GitHub Discussions](https://github.com/kero168/skillcheck/discussions) if
  enabled, otherwise an issue is fine.
- **Security problem** — never a public issue; see [SECURITY.md](SECURITY.md).

## Response expectations

This is a volunteer-maintained project. Issues are typically triaged within a
week; there is no SLA. Clear, minimal reproductions get answered fastest.

## Self-serve checklist

1. Update: `npm install -g skillcheck@latest`.
2. Run with `--json` to see exactly which rule fired and where.
3. Run `skillcheck rules` to read what a rule checks.
4. Check [README](README.md) / [README (日本語)](README.ja.md) and
   [docs/quickstart.md](docs/quickstart.md).
